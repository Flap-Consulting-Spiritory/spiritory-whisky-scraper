Production Setup Guide

Guía de Configuración
& Producción

Configura el scraper desde cero — desde el .env hasta el cron corriendo en background

Paso a paso — Windows
1
Requisitos previos
Instala Python 3.12+ y Git si no los tienes
ℹ️ Descarga Python desde python.org/downloads — marca la opción "Add Python to PATH" durante la instalación.
CMD / PowerShell
# Verificar versiones
python --version
git --version
2
Clonar el repositorio y crear entorno virtual
Clona el proyecto y activa el entorno Python aislado
CMD
# Clonar repositorio (ajusta la URL a la tuya)
git clone https://github.com/AlejandroSIlvaRodriguez/spiritoni-whisky-scraper.git
cd spiritoni-whisky-scraper

# Crear y activar entorno virtual
python -m venv venv
venv\Scripts\activate

# Instalar dependencias
pip install -r requirements.txt

# Instalar Chromium para Playwright
playwright install chromium
3
Crear y configurar el archivo .env
Copia el ejemplo y completa tus credenciales
CMD
copy .env.example .env
notepad .env
Contenido de .env
STRAPI_BASE_URL=https://strapi.spiritory.com/api
STRAPI_API_KEY=tu_api_key_de_strapi

VENICE_ADMIN_KEY=tu_api_key_de_venice

WHISKYBASE_USERNAME=tu_usuario_de_whiskybase
WHISKYBASE_PASSWORD=tu_password_de_whiskybase

# Opcional — hora UTC en que corre el cron (default: medianoche)
CRON_HOUR=0
CRON_MINUTE=0
Variable Descripción Tipo
STRAPI_BASE_URL URL base de tu API de Strapi Requerido
STRAPI_API_KEY Token Bearer de Strapi (Settings → API Tokens) Requerido
VENICE_ADMIN_KEY API Key de Venice AI para generar descripciones Requerido
WHISKYBASE_USERNAME Usuario de tu cuenta WhiskyBase Requerido
WHISKYBASE_PASSWORD Contraseña de tu cuenta WhiskyBase Requerido
CRON_HOUR Hora UTC de ejecución diaria (0–23) Opcional
CRON_MINUTE Minuto de ejecución diaria (0–59) Opcional
4
Guardar sesión de WhiskyBase
Abre Chrome, inicia sesión manualmente y guarda las cookies. Renueva cada 7–30 días.
CMD
# Asegúrate de que el venv está activado
venv\Scripts\activate

python save_wb_session.py
⚠️ Se abrirá una ventana de Chrome real. Inicia sesión en WhiskyBase, luego vuelve a la terminal y presiona Enter. Las cookies se guardan en wb_session.json.
5
Verificar que todo funciona
Tests rápidos antes de ir a producción
CMD
# Test 1: Conectividad con Strapi
python -c "from dotenv import load_dotenv; load_dotenv(); from integrations.strapi import fetch_bottles; b=fetch_bottles(limit=3); print(f'Strapi OK: {len(b)} botellas')"

# Test 2: Venice AI
python -c "from dotenv import load_dotenv; load_dotenv(); from utils.gemini import generate_description; print(generate_description('Smoky, vanilla finish.', 'Test Whisky'))"

# Test 3: Imports limpios
python -c "from scraper_engine import run_scraper; print('scraper_engine OK')"
python -c "from cron_daily import run_cron_cycle; print('cron_daily OK')"
6
Correr el cron en background (producción)
En Windows se usa start /B o una tarea programada de PowerShell
💡 En Windows no existe nohup. Usa start /B para correrlo desatachado, o créalo como Tarea Programada del sistema.
CMD — background inmediato
# Corre en background y redirige output a logs\cron.log
start /B python cron_daily.py > logs\cron.log 2>&1
PowerShell — background con Start-Process
# Opción recomendada — ventana propia y log separado
Start-Process python -ArgumentList "cron_daily.py" `
  -RedirectStandardOutput "logs\cron.log" `
  -RedirectStandardError "logs\cron_err.log" `
  -WindowStyle Hidden
CMD — test inmediato (dispara ahora)
# Útil para verificar que procesa correctamente
python cron_daily.py --run-now
CMD — ver el log en vivo
powershell -Command "Get-Content logs\cron.log -Wait -Tail 30"
PowerShell — detener el cron
# Buscar el PID del proceso
Get-Process python

# Detener por PID (reemplaza 1234 con el PID real)
Stop-Process -Id 1234
✅ El cron corre una vez al día a medianoche UTC. Cambia la hora con --hour 6 --minute 30 o via las variables CRON_HOUR / CRON_MINUTE en el .env.

Tests rápidos antes de ir a producción

# Test 1: Conectividad con Strapi

6
Correr el cron en background (producción)
Usa nohup para que siga corriendo aunque cierres la terminal
bash — iniciar en background
# Producción — corre indefinidamente, sobrevive al cierre de terminal
nohup python cron_daily.py > logs/cron.log 2>&1 &

# Guarda el PID para poder detenerlo después
echo $! > logs/cron.pid
echo "Cron iniciado con PID $(cat logs/cron.pid)"
bash — test inmediato (dispara ahora)
# Útil para verificar que procesa correctamente
python cron_daily.py --run-now
bash — ver log en tiempo real
tail -f logs/cron.log
bash — detener el cron limpiamente
# SIGTERM — el daemon termina la botella actual y sale
kill -TERM $(cat logs/cron.pid)

# Verificar que se detuvo
ps aux | grep cron_daily
bash — hora personalizada
# Correr a las 6:30 AM UTC
nohup python cron_daily.py --hour 6 --minute 30 > logs/cron.log 2>&1 &
✅ El cron corre una vez al día a medianoche UTC. El daemon termina siempre la botella en proceso antes de parar — nunca deja datos a medias.
