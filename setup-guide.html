Setup & Production
Guide

Configure the scraper from scratch — from .env to the cron running in the background

Step by step — Windows
1
Prerequisites
Install Python 3.12+ and Git if you don't have them
ℹ️ Download Python from python.org/downloads — check "Add Python to PATH" during installation.
CMD / PowerShell
# Verify versions
python --version
git --version
2
Clone the repository and create a virtual environment
Clone the project and activate the isolated Python environment
CMD
# Clone repository (adjust the URL to yours)
git clone https://github.com/AlejandroTechFlap/spiritory-whisky-scraper.git
cd spiritoni-whisky-scraper

# Create and activate virtual environment
python -m venv venv
venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Install Chromium for Patchright
patchright install chromium
3
Create and configure the .env file
Copy the example and fill in your credentials
CMD
copy .env.example .env
notepad .env
.env contents
STRAPI_BASE_URL=https://strapi.spiritory.com/api
STRAPI_API_KEY=your_strapi_api_key

VENICE_ADMIN_KEY=your_venice_api_key

# Optional — UTC time when the cron runs (default: midnight)
CRON_HOUR=0
CRON_MINUTE=0
ℹ️ No WhiskyBase credentials needed. patchright patches Chromium's fingerprint at the C++ level to bypass Cloudflare — WhiskyBase serves review content without authentication.
Variable Description Type
STRAPI_BASE_URL Base URL of your Strapi API Required
STRAPI_API_KEY Strapi Bearer token (Settings → API Tokens) Required
VENICE_ADMIN_KEY Venice AI API key for generating descriptions Required
CRON_HOUR UTC hour for daily run (0–23) Optional
CRON_MINUTE Minute for daily run (0–59) Optional
4
Verify everything works
Quick tests before going to production
CMD
# Test 1: Strapi connectivity
python -c "from dotenv import load_dotenv; load_dotenv(); from integrations.strapi import fetch_bottles; b=fetch_bottles(limit=3); print(f'Strapi OK: {len(b)} bottles')"

# Test 2: Venice AI
python -c "from dotenv import load_dotenv; load_dotenv(); from utils.gemini import generate_description; print(f'Venice AI OK: {generate_description(\"Smoky, vanilla finish.\", \"Test Whisky\")}')"

# Test 3: Clean imports
python -c "from scraper_engine import run_scraper; print('scraper_engine OK')"
python -c "from cron_daily import run_cron_cycle; print('cron_daily OK')"

# Test 4: WhiskyBase Scrape
python -c "from dotenv import load_dotenv; load_dotenv(); from integrations.whiskybase import scrape_bottle_data, close_session; print('Extrayendo datos de la botella 24413...'); data=scrape_bottle_data('24413'); print(f'\nWhiskyBase OK! Datos obtenidos:\n{data}'); close_session()"
5
Run the cron in the background (production)
On Windows use start /B or a PowerShell scheduled task
💡 On Windows there is no nohup. Use start /B to detach, or create it as a Windows Scheduled Task.
CMD — background
# Run in background, redirect output to logs\cron.log
start /B python cron_daily.py > logs\cron.log 2>&1
PowerShell — background with Start-Process
# Recommended option — separate window and log file
Start-Process python -ArgumentList "cron_daily.py" `
  -RedirectStandardOutput "logs\cron.log" `
  -RedirectStandardError "logs\cron_err.log" `
  -WindowStyle Hidden
CMD — immediate test (fires now)
# Useful to verify correct processing
python cron_daily.py --run-now
CMD — watch live log
powershell -Command "Get-Content logs\cron.log -Wait -Tail 30"
PowerShell — stop the cron
# Find the process PID
Get-Process python

# Stop by PID (replace 1234 with the real PID)
Stop-Process -Id 1234
✅ The cron runs once a day at midnight UTC. Change the time with --hour 6 --minute 30 or via CRON_HOUR / CRON_MINUTE in the .env.

5
Run the cron in the background (production)
Use nohup so it keeps running after closing the terminal
bash — start in background
# Production — runs indefinitely, survives terminal close
nohup python cron_daily.py > logs/cron.log 2>&1 &

# Save the PID so you can stop it later
echo $! > logs/cron.pid
echo "Cron started with PID $(cat logs/cron.pid)"
bash — immediate test (fires now)
# Useful to verify correct processing
python cron_daily.py --run-now
bash — watch live log
tail -f logs/cron.log
bash — stop the cron cleanly
# SIGTERM — daemon finishes the current bottle then exits
kill -TERM $(cat logs/cron.pid)

# Verify it stopped
ps aux | grep cron_daily
bash — custom time
# Run at 6:30 AM UTC
nohup python cron_daily.py --hour 6 --minute 30 > logs/cron.log 2>&1 &
✅ The cron runs once a day at midnight UTC. The daemon always finishes the bottle in progress before stopping — it never leaves partial data.
